# 第一章 P0 美术资产生产规范

## 1. 目标

本文档用于指导《同舟喵济》第一章「AI 与协议革命」的 P0 可上线美术资产生产。

本阶段目标不是一次性精修全部美术，而是先形成一套可批量生产、可索引、可接入游戏的资产规范，保证后续从游戏 UI、地图、角色、怪物、道具到细节 props 的 image 生产不会散乱。

P0 的定义：

- 可以支撑玩家完成「进入第一章 -> 多张中小地图探索 -> 遭遇怪物 -> 自走战斗 -> 获得道具/学分 -> 挑战最终 Boss -> 通关」的完整闭环。
- 资产风格统一，命名清晰，路径稳定。
- 每批生成的素材必须同步生成使用指引 MD，方便未来查找、复用和程序接入。
- 地图资产必须遵循「概念整图先行 -> 拆分背景/瓦片/props -> 重新拼装 -> 与概念图高度一致」的工作流。

## 2. 第一章地图结构

第一章不再采用一张超大地图，而采用 3-4 张中小地图互相传送的结构。

P0 推荐使用 4 张小地图：

| 地图 ID | 地图名称 | 场景类型 | 核心功能 | 主要内容 |
| --- | --- | --- | --- | --- |
| `ch1_m01_classroom_spawn` | 课程教室/出生地 | 室内 | 新手教学、出生点 | 陆教授、选课确认台、教学课件、拖延弹窗 |
| `ch1_m02_prompt_archive` | 提示词走廊与资料室 | 室内 | 提示词、隐私、引用、幻觉教学 | 含糊需求怪、复制粘贴影、资料桌、展板、图鉴点 |
| `ch1_m03_agent_lab` | 路演教室与机房 | 室内 | 中段战斗压力、章节小 Boss | 幻觉答案体、过拟合史莱姆、需求膨胀兽、投影墙、机房设备 |
| `ch1_m04_library_lawn_boss` | 图书馆外草坪 | 室外 | 章节收束、最终 Boss | 图书馆外立面、草坪、步道、Boss 考核区、AI 陆教授 |

如果制作压力较大，可以将 `ch1_m02_prompt_archive` 与 `ch1_m03_agent_lab` 合并，形成 3 张小地图版本。

地图连接关系：

```text
+------------------+      +------------------------+
| 课程教室/出生地 | ---> | 提示词走廊与资料室     |
+------------------+      +------------------------+
                                  |
                                  v
                         +------------------------+
                         | 路演教室与机房         |
                         +------------------------+
                                  |
                                  v
                         +------------------------+
                         | 图书馆外草坪/Boss      |
                         +------------------------+
```

传送点要求：

- 每张地图至少有 1 个入口和 1 个出口。
- 出入口必须在概念图、最终拼装图和使用指引 MD 中标明。
- 传送点可以表现为门、楼梯、走廊尽头、玻璃门、草坪入口或 Boss 考核门。
- 传送触发区需要记录建议坐标、朝向、目标地图 ID 和目标出生点。

## 3. 地图 image 生产总流程

所有第一章地图都按同一流程生产。

### 3.1 概念整图先行

每张地图必须先生成一张「整图概念图」。

概念整图用途：

- 确定整体构图、空间动线、主视觉锚点和交互点。
- 作为后续拆分地面、背景、props 和最终拼装的唯一视觉基准。
- 用于评估这张地图适合瓦片化，还是适合直接使用干净背景图。

概念整图要求：

- 俯视或 3/4 俯视 2D RPG 地图视角。
- 画面中不能出现 UI、文字水印、Logo、水印签名或无关说明字。
- 主要通路要清晰，不能被复杂装饰遮挡。
- 重要交互物件必须可识别，包括 NPC 点、怪物点、资源点、解密点、出入口。
- 室内三图需要共享同一套课程楼风格；最终 Boss 草坪图可以独立成室外风格，但角色比例、色彩饱和度和光照应与前三图兼容。

### 3.2 决定地面方案

概念整图确认后，必须判断该地图使用哪种地面方案。

| 方案 | 适用场景 | 输出内容 | 备注 |
| --- | --- | --- | --- |
| 瓦片图方案 | 地面重复规律明确，墙体/地板/草坪/步道可复用 | tileset、tilemap JSON、assembled preview | 默认优先方案，便于后续编辑 |
| 干净背景图方案 | 背景复杂、透视关系强、细节难以瓦片化 | clean background PNG、props PNG、collision mask、assembled preview | 允许 P0 直接使用，但必须保持通路干净 |
| 混合方案 | 地面可瓦片化，但墙面/远景/图书馆立面复杂 | tileset、局部背景层、props PNG、assembled preview | 推荐给图书馆外草坪和复杂室内图 |

默认 tile size 为 `64 x 64 px`。如果后续程序另定规格，以程序规格为准，但使用指引 MD 必须记录实际 tile size。

### 3.3 拆分背景与地面

根据概念整图生成地面或背景资产。

瓦片图方案 P0 输出：

- 地面 tileset。
- 墙面/边界 tileset。
- 可行走区域 tile。
- 不可行走区域 tile。
- 转角、边缘、阴影、过渡 tile。
- tilemap JSON。
- 地图 assembled preview。

干净背景图方案 P0 输出：

- 无角色、无怪物、无 UI、无掉落物的 clean background。
- 与 clean background 对齐的 collision mask 或碰撞说明图。
- 地图 assembled preview。
- 关键区域坐标说明。

### 3.4 props 单图生产

地上 props 不直接烙死在最终图里，除非该图明确采用完全静态背景方案。

props 输出要求：

- 每个可复用 prop 输出为独立透明 PNG。
- 同一类 prop 可以先做 atlas，但使用指引中必须列出每个 prop 的切片范围或坐标。
- 每个 prop 必须记录用途、建议锚点、碰撞建议和所在地图。
- 有互动功能的 prop 必须单独命名，不能只作为装饰写入背景。

P0 props 类型：

| 类型 | 示例 |
| --- | --- |
| 室内教学 | 讲台、课桌、椅子、白板、投影幕、电脑、资料柜、签到桌 |
| 走廊与资料室 | 展板、资料桌、文件堆、引用卡片、隐私提示牌、门牌 |
| 机房与路演 | 服务器机柜、屏幕墙、路演桌、投影光斑、智能体终端 |
| 图书馆外草坪 | 草坪块、石板路、长椅、路灯、告示牌、图书馆外墙局部、Boss 考核标记 |
| 交互/传送 | 门、楼梯、地图出口光圈、考核门、传送箭头占位 |

### 3.5 最终拼装必须回到概念图

每张地图必须产出一张最终 assembled preview。

assembled preview 要求：

- 使用实际 clean background、tileset、props 重新拼出来。
- 与概念整图的构图、道路、主要 props、视觉锚点几乎一致。
- 玩家出生点、怪物点、NPC 点、资源点、图鉴点、出口点必须在拼装图中可定位。
- 如果因瓦片限制导致与概念图不一致，必须在使用指引 MD 中说明调整原因。

验收标准：

- 重要视觉锚点位置基本一致，偏差不应超过 1 个 tile。
- 主通路宽度足够角色通行，不被 props 遮挡。
- 交互物件不与墙体、桌椅、草丛等不可行走物体重叠。
- 室内图之间的地板、墙体、光照和角色比例一致。
- 图书馆外草坪与室内图存在章节收束感，但不能像另一个游戏。

## 4. 第一章 P0 资产范围

P0 整体资产量建议控制在约 `110-140` 件。

如果复用现有五角色、部分 UI 和通用特效，第一章新增专属资产可控制在约 `75-100` 件。

| 模块 | P0 数量 | 说明 |
| --- | ---: | --- |
| 地图概念整图 | 4 | 每张小地图 1 张概念整图 |
| 地图背景/瓦片/拼装 | 16-28 | 每图背景或 tileset、tilemap、碰撞说明、assembled preview |
| 地图 props | 30-45 | 室内 props 与草坪 props，优先可复用 |
| 传送与交互标记 | 8-12 | 门、出口、考核门、互动高亮、图鉴点等 |
| 角色/NPC | 6-12 | 复用五主角时主要补陆教授、同小智、AI 陆教授表现 |
| 怪物与 Boss | 7-10 | 6 个普通/小 Boss 怪 + AI 陆教授 Boss |
| 技能与战斗特效 | 15-25 | 普攻、暴击、减速、DOT、护盾、Boss 三阶段技能、警戒/追击 |
| 道具/材料/增益装备 | 18-28 | 第一章道具、材料、宝箱、增益装备图标和品质框 |
| UI/HUD/结算 | 18-25 | 资源栏、学分、血条、行动条、技能槽、背包、结算、Boss 面板 |
| 事件/图鉴/解密 | 8-12 | 随机事件图标、2 个校史图鉴徽章、解密点图标 |
| 音效/BGM | 6-10 | 地图、Boss、攻击、受击、掉落、胜败、传送 |

## 5. P0 生产批次

建议按以下批次生产，避免素材先后依赖混乱。

| 批次 | 内容 | 产出 |
| --- | --- | --- |
| Batch 01 | 全局 UI 与资源图标 | HUD、按钮、背包、品质框、结算页、通用图标 |
| Batch 02 | 第一章地图风格基准 | 4 张地图概念整图、统一风格说明 |
| Batch 03 | 室内地图背景/瓦片 | 前 3 张室内小地图的 tileset 或 clean background |
| Batch 04 | 室内 props | 教室、走廊、资料室、机房、路演教室 props |
| Batch 05 | 图书馆外草坪 | Boss 地图概念、背景/瓦片、草坪 props |
| Batch 06 | 怪物与 Boss | 6 个怪物、AI 陆教授、必要头像或 sprite |
| Batch 07 | 技能与战斗表现 | 技能特效、命中特效、警戒圈、追击线、血条状态 |
| Batch 08 | 道具/装备/事件/图鉴 | 道具图标、增益装备、事件图标、图鉴徽章 |
| Batch 09 | 拼装与索引整理 | assembled previews、点位表、素材使用指引总索引 |

每个批次都必须生成一个对应的使用指引 MD。

## 6. 推荐目录结构

第一章地图资产和可复用游戏资产必须分开存放：

```text
assets/

    maps/

        background/
        chunks/
        foreground/
        props/
      ch1_m02_prompt_archive/
      ch1_m03_agent_lab/
      ch1_m04_library_lawn_boss/
  game/
    bosses/
    characters/npcs/
    enemies/
      animated/
      concepts/
      cutouts/
      portraits/
    vfx/
  ui/hud/
docs/
  asset-guides/
    chapter1-p0-asset-index.md
```

地图文件夹只允许存放地图图层。怪物、NPC、Boss、VFX 和 UI 必须放入可跨地图复用的公共目录；生成源图、QA 图和废弃版本不得放回运行资产目录。完整规则见 `assets/README.md`。

## 7. 命名规范

文件名使用英文小写、数字和连字符。

基础格式：

```text
ch1-[module]-[scene-or-object]-[variant]-v[number].png
```

示例：

```text
ch1-map-classroom-spawn-concept-v1.png
ch1-map-classroom-spawn-clean-bg-v1.png
ch1-map-classroom-spawn-tileset-v1.png
ch1-prop-classroom-lectern-v1.png
ch1-prop-archive-privacy-board-v1.png
ch1-enemy-vague-demand-icon-v1.png
ch1-boss-ai-professor-sprite-v1.png
ch1-ui-credit-meter-v1.png
ch1-item-prompt-card-v1.png
```

资产 ID 使用下划线：

```text
ch1_prop_classroom_lectern
ch1_enemy_vague_demand
ch1_map_library_lawn_boss
```

版本规则：

- `v1` 表示第一次可接入版本。
- 同路径替换美术但不改变尺寸和切片规则时，递增版本号。
- 如果改变 sprite sheet 切片、tile size、锚点或碰撞，需要在使用指引 MD 中明确标记为 breaking change。

## 9. 总索引 MD 要求

除每批使用指引外，还需要维护一个总索引：

```text
docs/asset-guides/chapter1-p0-asset-index.md
```

总索引至少包含：

- 第一章全部资产批次列表。
- 每个批次的使用指引 MD 路径。
- 每张地图的概念图、最终拼装图、背景/瓦片、props 路径。
- 每个怪物、Boss、道具、UI、特效的 asset_id 与文件路径。
- 当前可接入状态。
- 被废弃或替换的旧版本。

推荐总索引表：

```markdown
| asset_id | 名称 | 类型 | 文件路径 | 使用指引 | 状态 | 当前版本 | 备注 |
```

## 10. 地图验收清单

每张地图交付前必须检查：

- [ ] 有概念整图。
- [ ] 有最终拼装图。
- [ ] 已说明使用 tileset、clean background 还是 mixed。
- [ ] 主要视觉锚点与概念整图基本一致。
- [ ] 出生点、出口点、NPC、怪物、资源、图鉴点坐标已记录。
- [ ] props 如果可复用，已导出为透明 PNG。
- [ ] 不可行走区域和碰撞建议已记录。
- [ ] 地图没有水印、无关文字、错乱透视或遮挡主路的问题。
- [ ] 地图风格与第一章其它地图兼容。

## 11. UI 与图标验收清单

UI 与图标交付前必须检查：

- [ ] 资源图标能在小尺寸下识别。
- [ ] 品质边框颜色与白、绿、蓝、紫、橙、红六档对应。
- [ ] 按钮、面板、背包格、技能槽、结算页风格统一。
- [ ] 图标没有复杂背景，能放入背包、奖励结算和弹窗。
- [ ] UI 图片不内嵌不可替换的长文本。
- [ ] 同类图标尺寸一致。

## 12. 怪物与 Boss 验收清单

怪物与 Boss 交付前必须检查：

- [ ] 6 个普通/小 Boss 怪能一眼区分。
- [ ] AI 陆教授 Boss 与普通陆教授有明显差异，但仍能看出是课程系统镜像。
- [ ] 怪物图标可用于战斗结算页。
- [ ] 如果输出 sprite sheet，必须记录帧宽、帧高、帧序和动作名。
- [ ] 所有怪物的技能视觉方向与 GDD 配置匹配。
- [ ] 重要攻击特效不会遮挡血条和行动条。

## 13. P0 不做或暂缓内容

为了控制第一章上线范围，以下内容 P0 暂缓：

- 每个装备改变角色外观。
- 每个怪物制作完整 8 帧以上动画。
- 每个随机事件单独制作大幅插画。
- 每张地图都制作复杂动态前景。
- 大量稀有掉落专属动画。
- 复杂昼夜、天气、光照切换。
- 所有 NPC 的表情差分。

P0 优先保证完整闭环、风格统一、路径清楚、可接入。

## 14. 后续执行原则

后续批量 image 生产时，每个任务应先写清楚：

1. 要生成哪个批次。
2. 该批次要服务哪个地图、UI 或系统。
3. 需要参考哪张概念图或风格图。
4. 输出哪些 PNG、JSON 或 MD。
5. 使用指引 MD 的路径。
6. 这些素材是否可以直接接入游戏。

任何没有进入使用指引 MD 和总索引 MD 的图片，都视为临时素材，不能作为正式 P0 资产接入。
