## 部署

1. 停止旧服务，并备份线上 `play-data.sqlite3*` 数据文件。
2. 将压缩包解压到新的发布目录。不要用包内文件覆盖线上数据库；本包已排除本地数据库、日志、Git 元数据和临时 QA 文件。
3. 如需保留账号与角色数据，把备份的 `play-data.sqlite3`、`play-data.sqlite3-wal`、`play-data.sqlite3-shm` 放回发布目录。
4. 启动：`python play-server.py`。默认监听 `8787`，可使用环境变量 `PORT` 修改。
5. Nginx/阿里云反向代理需把 HTTP 与 WebSocket `/ws` 一并转发到该 Python 服务，并允许 MP4 Range 请求。该项目不是纯静态站点，不能只上传到静态对象存储。
6. 验证：`/play.html` 返回 200；未登录 `/api/me` 返回 401；`assets/cg/p1boss-end.mp4` 的 Range 请求返回 206。
